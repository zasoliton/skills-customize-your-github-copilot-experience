# Search Algorithms: Linear and Binary Search

## 🎯 Objective

Implement and compare linear and binary search algorithms, measure simple step counts or timings, and explain trade-offs.

## 📝 Tasks

### 🛠️ Implement search functions

#### Description
Write two functions in `starter-code.py`:

- `linear_search(arr, target)` — search `arr` for `target` using a linear scan; return the index or `-1`.
- `binary_search(arr, target)` — search a sorted `arr` using binary search; return the index or `-1`.

#### Requirements

- Both functions must return the index of the found element or `-1` if missing.
- `binary_search` must assume the input array is sorted in ascending order.

### 🛠️ Compare performance

#### Description
Use the provided `benchmark_sample()` helper in `starter-code.py` or write a short script to compare the number of steps (loop iterations) or elapsed time for each algorithm on small arrays (e.g., 10, 100, 1000 elements).

#### Requirements

- Provide a short table or printed output describing when binary search is better and why.
- Discuss one case where linear search is preferable.

### 📤 Submission

- Submit the completed `starter-code.py` and a short text file or comment in the README describing your findings (1–3 paragraphs).
